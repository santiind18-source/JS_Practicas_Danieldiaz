8. Herencia Prototípica

8.1 Herencia Prototípica

Este concepto fundamental en JavaScript explica cómo los objetos pueden heredar propiedades y métodos de otros objetos, formando una "cadena de prototipos". Cuando se accede a una propiedad que no está directamente en un objeto, JavaScript busca sucesivamente en su prototipo, y luego en el prototipo de este, hasta llegar a null.

8.2 El Prototipo de Función Constructora (F.prototype)

En JavaScript, cada función constructora (por ejemplo, function Usuario() {...}) posee una propiedad especial prototype (F.prototype). Este objeto es el lugar idóneo para definir los métodos que serán compartidos y heredados por todas las instancias creadas con dicha función constructora mediante la palabra clave new. Esto optimiza el uso de memoria y establece la herencia.

8.3 Prototipos Nativos

JavaScript incluye objetos y constructores "nativos" (incorporados) como Object, Array, Function, String, Number, Date, entre otros. Cada uno dispone de su propio prototipo (ej., Array.prototype), el cual contiene los métodos y propiedades por defecto utilizados por sus instancias (ej., .push() para arrays). Es posible extender estos prototipos, aunque se debe hacer con extrema precaución para evitar conflictos.
